# 工作愿景

1. 第一人称，不是别人对你的总结

```
改前：功成身退，构建可复制的商业模式……
改后：我一直在思考如何从业务中"功成身退"……
```

愿景必须是"我"说出来的话，不是别人替你写的简介。只有第一人称才能承载真正的承诺。

## 2. 每段都有一个"可记住的核心句"

- 战略：做一个可以随时"离场"的系统
- 教育：让学习在"不经意"间发生
- 组织：建立一个不需要"指令"的局域网
- 新媒体：真实是筛选同类的最高杠杆

不是一堆形容词的堆砌，而是一句别人听了能记住、你自己能复述的话。

## 3. 通过否定来定义自己

好的愿景不是空洞的肯定，而是清楚地说出"我不做什么"：

- "不是为了逃避，而是为了验证"
- "我不信奉灌输，我信奉设计"
- "我不喜欢发号施令"
- "我不打算阉割那些被视为杂质的情绪"

边界比方向更重要。

## 4. 回答"为什么"，不是"做什么"

每段都在解释内在动机：

- 为什么要功成身退？→ 验证商业模式是否可复制
- 为什么要设计滑梯？→ 让学习者相信自己能学会任何东西
- 为什么要建立上下文共识？→ 让管理成为多余的噪音

没有"为什么"的愿景只是待办清单。

## 5. "原则性的具体"

既不是空洞的口号（"追求卓越"），也不是操作手册（"每周开一次会"）。而是在中间层：

> "当大家都在同一个认知维度里观察世界，所谓的'管理'就成了多余的噪音。行动不再源于顺从，而源于同频后的自发涌现。"

这足够抽象，可以指导多种场景；又足够具体，读完知道该怎么做。

## 6. 自带检验标准

每段都隐含了自己的验证方式：

| 愿景 | 检验标准 |
|------|----------|
| 随时离场 | 我离开一个月，公司还能正常运转吗？ |
| 不经意学习 | 学习者是否相信自己能学会任何东西？ |
| 认知局域网 | 团队是否不再需要我发号施令？ |
| 真实筛选 | 内容是否推开了错误的人、吸引了对的人？ |

没有检验标准的愿景是无法执行的。

---

总结一句话：**合格的工作愿景是"我"用"第一人称"说出的"有边界的承诺"，包含"为什么"，停留在"原则性的具体"层次，并且自带"检验标准"
